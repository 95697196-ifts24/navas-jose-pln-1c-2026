Módulo 1 — Adquisición de corpus 002_adquisicion_corpus/
🌐
Requests + BeautifulSoup
requests · bs4
Descarga HTML de páginas estáticas y navega el DOM para extraer texto, enlaces y metadatos. Ideal para sitios sin JavaScript dinámico.
scraping estático DOM / CSS selectors
En TPI 2 Útil para medios con HTML simple (Cenital, Anfibia, Chequeado). Permite extraer lista de URLs de una sección o autor. 
import requests
from bs4 import BeautifulSoup
r = requests.get("https://cenital.com/...")
soup = BeautifulSoup(r.text, "html.parser")
texto = soup.find("article").get_text()
📄
Trafilatura
trafilatura
Extrae el texto principal de un artículo web descartando automáticamente menús, publicidades y elementos de navegación. Es la herramienta más limpia para medios periodísticos.
extracción limpia periodismo digital recomendada TPI 2
En TPI 2 — la más recomendada Una sola llamada por URL. Devuelve texto limpio listo para el corpus. Compatible con Cenital, Anfibia, Infobae, La Nación. 
import trafilatura
descargado = trafilatura.fetch_url(url)
texto = trafilatura.extract(
    descargado,
    include_comments=False,
    favor_precision=True,
)
Limitación Puede fallar en sitios con mucho JavaScript. Combinar con Playwright si es necesario.
🎭
Playwright
playwright
Controla un browser real (Chromium/Firefox) para páginas dinámicas que cargan contenido con JavaScript. Permite esperar a que los elementos estén presentes antes de extraer.
scraping dinámico JavaScript
En TPI 2 Usar solo si Trafilatura falla en el medio elegido. Más lento pero más robusto ante sitios complejos.
Limitación Lento, requiere instalar browsers (~100 MB). Usar como fallback, no como primera opción.
🗂️
Pandas — armado del CSV
pandas · pathlib
Una vez extraídos los textos, se estructuran en un DataFrame con las columnas requeridas por la consigna y se exportan como CSV o JSONL.
estructura de datos CSV / JSONL
En TPI 2 — obligatorio El corpus debe tener: id · fecha · medio · autor · titulo · texto · grupo_comparacion. El notebook valida estas columnas automáticamente. 
import pandas as pd
df = pd.DataFrame(registros)
df.to_csv("corpus_tpi2.csv", index=False)
Módulo 2 — Preprocesamiento con spaCy 003_spacy/
✂️
Tokenización
spacy · nlp(text)
Divide el texto en unidades mínimas (tokens): palabras, signos de puntuación, espacios. Es el primer paso de cualquier pipeline NLP. spaCy aplica reglas lingüísticas para el español.
primer paso obligatorio
En TPI 2 Se aplica implícitamente al llamar nlp(texto). Genera el objeto Doc base para lematización, NER y todo el resto.
📝
Lematización
token.lemma_
Reduce cada palabra a su forma base: "comieron" → "comer", "políticas" → "política". Agrupa variantes flexivas para que los conteos no queden dispersos en formas distintas del mismo concepto.
normalización morfología
En TPI 2 Fundamental para que BoW y TF-IDF agrupen correctamente las variantes. Usar token.lemma_.lower() sobre el texto de contenido filtrado.
Limitación es_core_news_sm no lematiza bien el voseo rioplatense ("tenés" → "tener" puede fallar). Considerar un diccionario de corrección manual.
🚫
Filtrado de stopwords
token.is_stop · token.is_alpha
Descarta artículos, preposiciones y conjunciones ("el", "de", "que", "y") que aparecen en todos los textos sin aportar información temática. Reduce ruido en los conteos de frecuencia.
filtrado ruido léxico
En TPI 2 Usar token.is_alpha and not token.is_stop para construir el campo texto_contenido. Es la base para BoW y TF-IDF significativos. 
# Texto de contenido (sin ruido)
contenido = [
    token.lemma_.lower()
    for token in doc
    if token.is_alpha and not token.is_stop
]
🏷️
NER — Entidades Nombradas
doc.ents · ent.label_
Detecta y clasifica nombres propios: personas (PER), organizaciones (ORG), lugares (LOC) y misceláneos (MISC). Permite ver quiénes y qué instituciones estructuran el discurso de cada grupo.
personas organizaciones lugares
En TPI 2 — muy útil para comparar Las entidades revelan qué actores, medios y lugares nombra cada grupo. Una diferencia sistemática en NER puede ser un hallazgo discursivo central. 
for ent in doc.ents:
    print(ent.text, ent.label_)
# "OpenAI" MISC, "Argentina" LOC
Limitación es_core_news_sm comete errores frecuentes en entidades muy específicas o neologismos. Verificar manualmente los más relevantes.
⚙️
POS Tagging
token.pos_ · token.dep_
Etiqueta cada token con su categoría gramatical: NOUN, VERB, ADJ, ADV, PROPN, etc. Permite filtrar solo sustantivos o adjetivos para análisis de valoración o análisis temático más específico.
gramática morfología
En TPI 2 Opcional pero útil: filtrar solo sustantivos (NOUN) para observar el universo léxico de temas, o adjetivos (ADJ) para analizar valoración.
🔄
Pipelines de preprocesamiento
spaCy · Stanza · spacy-stanza
Diferentes estrategias producen diferentes representaciones: Pipeline A (naive: lowercase), Pipeline B (lemas + stopwords), Pipeline C (con correcciones manuales). La elección es una decisión metodológica que debe justificarse.
decisión metodológica reproducibilidad
En TPI 2 El notebook ya implementa el Pipeline B adaptado. Documentar qué modelo y versión se usa. Si hay mucho voseo, agregar correcciones al diccionario.
Módulo 3 — Vectorización sparse 005_Vectorizacion/ · 006_lab_integrador_guiado/
🎒
Bag of Words (BoW)
CountVectorizer — sklearn
Representa cada documento como un vector de conteos de palabras. Ignora el orden; solo importa la presencia y frecuencia. Produce una matriz documentos × términos donde cada celda es la cantidad de apariciones.
frecuencias absolutas volumen léxico obligatorio TPI 2
En TPI 2 — obligatorio Muestra qué palabras dominan el vocabulario de cada grupo por volumen. Combinar siempre con TF-IDF para distinguir volumen de especificidad. 
from sklearn.feature_extraction.text import CountVectorizer
vec = CountVectorizer(lowercase=False, min_df=1, max_df=0.95)
matriz = vec.fit_transform(corpus_por_grupo["texto"])
vocab = vec.get_feature_names_out()
Limitación Palabras muy frecuentes (aunque no sean stopwords) dominan la vista. No distingue si un término es importante o simplemente abundante.
⚖️
TF-IDF
TfidfVectorizer — sklearn
Pondera cada término por su frecuencia en el documento (TF) multiplicada por la rareza en el corpus completo (IDF). Las palabras comunes a todos los documentos bajan de peso; las específicas de cada grupo suben.
especificidad distintividad obligatorio TPI 2
En TPI 2 — obligatorio y clave Revela qué términos son más propios de cada grupo (no solo los más frecuentes). El heatmap de TF-IDF es la visualización más informativa para el análisis comparado. 
from sklearn.feature_extraction.text import TfidfVectorizer
vec = TfidfVectorizer(lowercase=False, min_df=1, max_df=0.95)
matriz = vec.fit_transform(corpus_por_grupo["texto"])
# Peso alto = término específico de ese grupo
Limitación Con corpus de 6-10 textos el IDF tiene poco poder discriminante. Los pesos son orientativos; siempre volver al fragmento para confirmar.
🔗
N-gramas (bigramas / trigramas)
CountVectorizer(ngram_range=(2,2))
Captura secuencias de N palabras consecutivas. Los bigramas (2 palabras) detectan colocaciones como "inteligencia artificial", "cambio climático" o "banco central" que los unigramas fragmentan.
colocaciones contexto obligatorio TPI 2
En TPI 2 — observables iniciales Los bigramas por grupo revelan asociaciones conceptuales que los términos sueltos ocultan. Un bigrama muy frecuente en un grupo puede ser un hallazgo discursivo en sí mismo. 
from sklearn.feature_extraction.text import CountVectorizer
vec_bi = CountVectorizer(
    lowercase=False,
    ngram_range=(2, 2),
    min_df=1
)
matriz_bi = vec_bi.fit_transform(df["texto_contenido"])
Limitación Con corpus pequeño, la mayoría de bigramas aparece solo 1 vez. Revisar manual y selectivamente los más relevantes.
📐
Similitud coseno
cosine_similarity — sklearn
Mide el ángulo entre dos vectores en el espacio de términos. Un valor cercano a 1 indica textos muy similares; cercano a 0, textos muy distintos. Funciona sobre matrices BoW y TF-IDF.
similitud comparación
En TPI 2 Opcional pero útil para argumentar que los dos grupos son efectivamente distintos. Una similitud baja entre grupos confirma la pertinencia del contraste elegido.
COMPARACIÓN BoW vs TF-IDF
Aspecto Bag of Words (BoW) TF-IDF
Qué mide Frecuencia absoluta de cada término en el documento Importancia relativa: frecuente en ese doc pero raro en el resto
Muestra Qué palabras dominan en volumen Qué palabras son características y distintivas
Sesgo Favorece términos muy repetidos (incluso si aparecen en todos) Penaliza palabras omnipresentes; premia las específicas
Mejor para Ver temas centrales por peso bruto, detectar insistencias Identificar el vocabulario que vuelve singular a cada grupo
En análisis discursivo Muestra la agenda temática de cada grupo Muestra el registro propio de cada grupo
Limitación clave No discrimina importancia; "el" podría dominar sin stopwords Con corpus pequeño (6-10 docs), el IDF tiene poco poder
Módulo 4 — Visualización analítica matplotlib · seaborn
📊
Barras horizontales comparativas
sns.barplot(hue=grupo)
Compara los N términos más frecuentes de cada grupo lado a lado. Es la visualización más directa para la lectura distante: permite ver similitudes y diferencias de vocabulario de un vistazo.
frecuencias comparación directa
En TPI 2 — usar en sección 7 Una de las dos visualizaciones analíticas obligatorias. Mostrar top 10-12 términos por grupo, usando la misma escala de eje X para ambos grupos.
🌡️
Heatmap TF-IDF
sns.heatmap(cmap="mako")
Mapa de calor donde el color muestra el peso TF-IDF de cada término en cada grupo. Los términos blancos/claros son compartidos; los oscuros son exclusivos del grupo. Es muy efectivo para mostrar distintividad.
TF-IDF distintividad muy recomendado
En TPI 2 — sección 8, visualización clave Segunda visualización analítica recomendada. Muestra de golpe qué vocabulario es exclusivo de cada grupo versus compartido. 
sns.heatmap(
    matriz_heatmap,
    cmap="mako",
    linewidths=0.4,
    cbar_kws={"label": "peso TF-IDF"},
)
📦
Barras de longitud / balance
sns.barplot
Compara cantidad de documentos y palabras totales entre los dos grupos. Permite auditar si el corpus está balanceado o si un grupo domina en volumen, lo que condicionaría los resultados.
auditoría del corpus balance
En TPI 2 — sección 5 Ya está implementado en el notebook. Si hay desbalance, mencionarlo en la sección de límites del método.
📋
Tablas de fragmentos
pd.DataFrame · display()
Muestra citas textuales con contexto alrededor del término buscado. No es un gráfico, pero es una forma de visualizar la evidencia discursiva que justifica los hallazgos cuantitativos.
lectura cercana evidencia textual
En TPI 2 — sección 9, obligatorio Volver a al menos 3 fragmentos concretos. Esta "visualización" es lo que distingue un análisis discursivo de una simple estadística de palabras.
📈
Principios de diseño visual
sns.despine · constrained_layout
Criterios para gráficos legibles: paleta accesible (colorblind), ejes sin marcos redundantes (despine), títulos a la izquierda, etiquetas numéricas directas en las barras, sin leyendas innecesarias.
accesibilidad claridad
En TPI 2 Seguir la configuración del notebook: palette="colorblind", sns.despine(), títulos explicativos. La rúbrica evalúa "gráficos legibles y bien rotulados" (20%).
Módulo 5 — Representaciones semánticas 008_representaciones_semanticas/ — solo como marco conceptual en TPI 2
🧠
Vectores densos (spaCy md)
es_core_news_md · doc.similarity()
El modelo mediano de spaCy incluye vectores pre-entrenados de 50 dimensiones. Permiten calcular similitud semántica entre palabras o documentos capturando sinónimos y campos conceptuales que BoW no detecta.
semántica no permitido en TPI 2
Advertencia para TPI 2 La consigna prohíbe usar embeddings o vectores densos como sustituto del recorrido pedido (BoW + TF-IDF + spaCy + bigramas). Incluir esta técnica fuera del flujo pedido podría restarle validez a la entrega.
🔍
Limitaciones de BoW/TF-IDF
contexto conceptual
BoW y TF-IDF no capturan: (1) sinónimos ("papá" ≠ "padre"), (2) polisemia ("banco" institución vs asiento), (3) relaciones conceptuales entre campos temáticos, (4) orden y contexto oracional.
limitaciones sparse reflexión metodológica
En TPI 2 — sección de límites del método Estas limitaciones son exactamente lo que se debe mencionar en la reflexión metodológica (15% de la rúbrica). Argumentar con ejemplos concretos del propio corpus.
Módulo 6 — Estrategia de análisis discursivo 006_lab_integrador_guiado/ · Marco: Moretti · Maeda
🔭
Lectura distante
Franco Moretti
Analizar patrones a gran escala (frecuencias, distribuciones, comparaciones cuantitativas) sin leer cada texto en detalle. Permite detectar recurrencias y contrastes que no serían visibles desde la lectura individual.
escala macroscópica patrones
En TPI 2 — secciones 7 y 8 Los conteos, bigramas, BoW y TF-IDF son herramientas de lectura distante. La clave es identificar qué patrones justifican profundizar con lectura cercana.
🔬
Lectura cercana
extraer_fragmentos()
Volver al texto concreto una vez identificado un patrón cuantitativo. El fragmento permite ver el tono, el marco argumentativo y el contexto que los números no pueden capturar.
evidencia textual interpretación obligatorio TPI 2
En TPI 2 — sección 9, mínimo 3 fragmentos El retorno al fragmento es el momento donde la computadora deja de hablar y el analista toma la palabra. Sin este paso el trabajo no alcanza el mínimo.
💬
Reflexión metodológica
Sección de límites del método
Conciencia explícita de qué puede y qué no puede mostrar el enfoque elegido. Qué afirmaciones son sostenibles con la evidencia del corpus y cuáles exceden lo que las representaciones sparse permiten.
pensamiento crítico 15% rúbrica
En TPI 2 — sección final, 15% de la nota Mencionar: tamaño del corpus, limitaciones del IDF con pocos documentos, errores de NER, pérdida de contexto en BoW, sesgo de lematización.
